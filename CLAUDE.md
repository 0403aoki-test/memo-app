# CLAUDE.md

## コーディングルール

### Git
- `.env` ファイルは絶対にコミット・プッシュしない
- 機密情報（APIキー、パスワード、トークン）をコードに直書きしない
- `node_modules/` はコミットしない
- コミットメッセージは変更内容がわかるように書く
- 作業は必ずブランチを切って行う（mainに直接コミットしない）
- PRを経由してmainにマージする

### TypeScript
- `any` 型の使用は避ける
- 型は明示的に定義する
- `console.log` はデバッグ後に削除する

### ファイル・ディレクトリ
- `.gitignore` に以下を必ず含める
  - `.env`
  - `node_modules/`
  - ビルド成果物（`dist/`, `build/`）

### 環境変数
- 環境変数は `.env` ファイルで管理する
- `.env.example` にキー名だけ記載してコミットする（値は書かない）

### その他
- 依存パッケージは定期的にアップデートする
- 不要なファイルはこまめに削除する
