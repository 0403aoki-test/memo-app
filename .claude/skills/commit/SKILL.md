# Commit Skill

ユーザーが `/commit` を実行したとき、以下の手順で安全にコミットを行う。

## 引数

- `/commit` — 差分からコミットメッセージを自動生成する
- `/commit "メッセージ"` — 指定したメッセージでコミットする

## 手順

### 1. 変更内容の確認

以下のコマンドを並行して実行し、現状を把握する:

```bash
git status
git diff
git log --oneline -5
```

### 2. 安全チェック（必ず実施）

以下の条件をすべて確認する:

**コミットしてはいけないファイル:**
- `.env` / `.env.*`（環境変数・シークレット）
- `node_modules/`（依存パッケージ）
- `dist/`, `build/`（ビルド成果物）

**ブランチチェック:**
- 現在のブランチが `main` または `master` の場合は、必ずユーザーに確認してから進む

**機密情報チェック:**
- APIキー、パスワード、トークンなどがファイルに直書きされていないか確認する

### 3. ファイルのステージング

安全なファイルのみを `git add` でステージングする。  
危険なファイルは除外する。

```bash
# 個別にファイルを指定してステージング（git add -A や git add . は使わない）
git add <安全なファイルパス1> <安全なファイルパス2> ...
```

### 4. コミットメッセージの作成

引数でメッセージが渡された場合はそれを使う。  
引数がない場合は差分から以下の形式で自動生成する:

```
<種別>: <変更内容の要約>

例:
feat: ユーザー認証機能を追加
fix: ログインエラーのバグを修正
refactor: APIクライアントのコードを整理
docs: READMEにセットアップ手順を追記
style: Tailwind CSSのクラス名を整理
chore: 依存パッケージをアップデート
```

### 5. コミット実行

```bash
git commit -m "$(cat <<'EOF'
<コミットメッセージ>

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"
```

### 6. 結果確認

```bash
git log --oneline -1
```

コミット内容をユーザーに表示して完了を伝える。
